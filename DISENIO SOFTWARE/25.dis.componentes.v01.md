## 2. Componentes de Software

## 2.0 Diseño Objetivo (TO-BE)

La solución se organiza en tres módulos funcionales, expuestos como APIs REST y procesos asíncronos:

1. **Simulación Multi-día + Streaming:** inicia simulaciones y emite telemetría continua (SSE).
1. **Planificador (HGA/ALNS):** planifica y replanifica rutas en memoria.
1. **Experimentación Numérica (DoE) + Exportación:** ejecuta batches y genera Excel.

### 2.1 Backend — API REST Controllers (Capa de Presentación Spring)
Los controladores manejan el ciclo de vida de la red HTTP, parsean parámetros, y enrutan peticiones a los servicios asíncronos.

| Componente | Tipo Spring | Responsabilidad Real en Implementación |
|---|---|---|
| `SimulationController` | `@RestController` | Expone `/api/v1/simulation`. Contiene `POST /run/{dias}` para encolar nuevas tareas asíncronas y `GET /status/{sessionId}` para obtener reportes pesados, tablas y demanda (usado usualmente al término, cuando el estado es `DONE`). |
| `SimulationV2Controller` | `@RestController` | Expone la v2 optimizada (`/api/v2/simulation`). Emite `SseEmitter` (`/stream/{id}`) generando un flujo incesante de DTOs mapeados desde memoria hacia la capa cliente. Elimina la necesidad de Polling constante. |
| `NumericExperimentController` | `@RestController` | Permite gestionar ejecuciones Diseño de Experimentos (DoE). Incluye endpoints de descarga de data estática. |
| `PlanningController` | `@RestController` | Expone `/planificador` para planificación inicial (HGA/ALNS), replanificación por cancelación y consulta de la solución vigente. |

### 2.2 Backend — Servicios y Motor Interno (Capa de Negocio)
Constituyen el corazón transaccional y analítico del sistema logístico.

| Clase / Interface | Tipo Spring | Responsabilidad / Descripción Técnica |
|---|---|---|
| `SimulationService` | `@Service` | Ejecución multi-día asíncrona (`@Async("simulationExecutor")`). Carga datos por fecha, planifica (HGA/ALNS) y simula eventos; actualiza `SimulationProgressHolder` para consumo v1 y SSE v2. |
| `SimulationProgressHolder` | `@Component` | Mapa Concurrente en memoria (`ConcurrentHashMap`). Guarda instancias de `SimulationSessionState`. Actúa como "Base de Datos en Caliente" durante la simulación. Evita la saturación del I/O del motor SQL durante la ejecución. |
| `SimulationExcelService` | `@Service` | Utilitario que genera archivos binarios interactivos `.xlsx` desde los reportes almacenados en memoria al finalizar una ejecución algorítmica profunda. |
| `HGAPlannerService` | `@Service` | Motor HGA. Produce solución base y precalcula rutas de respaldo (warm-start) para replanificación ALNS. |
| `ALNSPlannerService` | `@Service` | Motor ALNS. Planifica desde cero (comparación DoE) o replanifica ante cancelación de vuelos usando warm-start. |
| `PlanningService` | `@Service` | Fachada del planificador: expone HGA/ALNS a controllers y administra el estado en `PlanningSessionHolder`. |
| `PlanningSessionHolder` | `@Component` | Memoria de trabajo del planificador (última `Solution` vigente). |
| `RouteBuilder` | `@Component` | Construye rutas usando `NetworkAdapter` (Dijkstra), con soporte de capacidad y rutas alternativas (backup). |
| `NetworkAdapter` | Interface | Abstracción del grafo en memoria: rutas óptimas y alternativas; permite invalidación del grafo. |

### 2.3 Frontend — Patrón Container-Presenter (React)
El frontend organiza su lógica delegando el "Estado y Lógica del Negocio UI" a Custom Hooks, dejando a los Componentes (Presenters) como meras funciones de pintado UI.

| Módulo / Archivo | Tipo Estructural | Responsabilidad |
|---|---|---|
| `api.js` | Helper Funcional | Centraliza la creación y gestión de `fetch` a la capa backend. Exporta utilidades como `apiUrl` para formatear los URIs (vital para SSE y CORS). |
| `useControlTowerController.js` | Custom Hook | Concentrador del estado principal. Inicia el `EventSource` contra la ruta v2, escucha eventos (`snapshot`, `kpi`, `eventLog`), y actualiza el objeto `liveStatus`. Intercepta estados terminales para cerrar el túnel SSE e iniciar llamadas `apiFetch` finales. |
| `useNumericExperiment.js` | Custom Hook | Coordina el progreso, estados de simulación y descarga de reportes asíncronos para el dashboard de experimentación (DoE). |
| `App.jsx` | Shell UI | Componente raíz de la Torre de Control. Ensambla paneles, escenarios, navegación a experimentación y mapa. |
| `WorldMap.jsx` | Presenter UI (SVG Map) | Recibe rutas activas y métricas y renderiza el mapa interactivo (react-simple-maps). |
| `AlgorithmComparisonPanel` | Presenter UI | Componente colapsable que renderiza métricas estáticas (ej. tiempos HGA vs ALNS). Recibe datos desde el Custom Hook en fase terminal. |

### 2.4 Patrones de Diseño Formales
1. **Server-Sent Events (Observer UI-to-Server):** `SimulationV2Controller` actúa como observable de un flujo de `SimulationSessionState`, y React a través de `EventSource` actúa como suscriptor puro. 
2. **State Holder (In-Memory Database):** `SimulationProgressHolder` utiliza Singleton/ConcurrentMap para crear zonas críticas aisladas por `sessionId`, logrando que las lecturas SSE no bloqueen a las escrituras del ALNS/HGA.
3. **Strategy (Algoritmos):** El servicio invocador selecciona dinámicamente si envía la carga de maletas al algoritmo estático HGA o al dinámico ALNS de acuerdo a los umbrales de configuración provistos por el usuario en React.

## 3. Flujo de Ejecución — Transmisión SSE
1. **Acción del Usuario:** El usuario en el componente de Torre de Control hace clic en "Ejecutar Simulación".
2. **Creación:** React invoca `POST /api/v1/simulation/run/5`.
3. **Inicio del Proceso:** Spring genera un UUID y lo devuelve inmediatamente (Código 202 Accepted). Tras bambalinas, un hilo `runAsync` inicializa a HGA en `SimulationService`.
4. **Suscripción SSE:** El Hook en React instancia `new EventSource('/api/v2/simulation/stream/{UUID}')`.
5. **Streaming Continuo:** Cada 500ms, el backend transfiere un objeto serializado (Snapshot y KPI) con coordenadas espaciales y porcentajes `airportLoads`. React los procesa sin hacer reload de página y actualiza el mapa.
6. **Cierre y Post-procesado:** Al recibir el estado `DONE` en la trama SSE, React llama explícitamente a `EventSource.close()`, y lanza un `GET` síncrono al Controlador v1 para descargar la sábana de Excel de resultados definitivos.
