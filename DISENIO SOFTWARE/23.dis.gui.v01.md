## 2. Diseño de Interfaz Gráfica de Usuario (GUI)
La interfaz de usuario del sistema adopta el paradigma **Dark Control Tower** (Torre de Control Oscura). Este diseño maximiza el contraste visual para resaltar los datos operativos críticos (Key Performance Indicators o KPIs) y facilitar la lectura en entornos de monitoreo prolongado, previniendo la fatiga visual de los analistas de rutas.

### 2.1 Patrón de Layout (Container-Presenter)
La aplicación web de una sola página (SPA) desarrollada en React (versión no relevante para el diseño) con Vite, utiliza un layout anclado y paneles flotantes que permiten maximizar el lienzo principal (el mapa).

| Zona Lógica | Implementación CSS / React | Responsabilidad |
|---|---|---|
| **Lienzo Central (Mapa)** | `react-simple-maps` (SVG Map) | Renderiza topológicamente los 30 nodos aeroportuarios oficiales (ICAO) usando coordenadas geoespaciales precisas. Contiene las aristas (rutas en curso) y las animaciones de tránsito. |
| **KPI Strip** | `kpiCards` (Array dinámico) | Franja superior que muestra indicadores vitales en tiempo real: Vuelos en curso, Ocupación global, Nivel de cumplimiento SLA (%) y Nodos críticos. |
| **Floating Rails** | Paneles flotantes laterales (AS-IS: composición en `App.jsx`) | Rieles de paneles translúcidos laterales. Albergan el Top de Aeropuertos, Inventario de Tránsito y Telemetría del Sistema. Se superponen sutilmente sobre el mapa utilizando *glassmorphism*. |
| **Dock Inferior** | `AlgorithmComparisonPanel` | Panel colapsable inferior dedicado a mostrar tablas comparativas del experimento numérico (HGA vs ALNS), ejecución de colapso y análisis post-simulación. |

### 2.2 Paleta de Colores y Micro-Interacciones
La aplicación prescinde de clases utilitarias genéricas (como las de Tailwind por defecto) y opta por Vanilla CSS o variables CSS inyectadas para tener control estricto de la identidad visual.

| Token CSS | Valor HEX | Uso Semántico |
|---|---|---|
| `--ct-bg-primary` | `#0B131E` | Fondo base de la aplicación. Tonos fríos oscuros. |
| `--ct-bg-panel` | `#1A2635` / `#16212D` | Fondos de paneles. Emplean transparencia (opacity/backdrop-filter) para crear profundidad. |
| `--ct-accent` | `#3B82F6` | Elementos de acción, tabs activos, focus rings. |
| `--ct-status-ok` | `#10B981` | Semáforo: Ocupación segura, SLA > 90%. |
| `--ct-status-warn` | `#F59E0B` | Semáforo: Ocupación > 70%, SLA en riesgo. |
| `--ct-status-critical` | `#EF4444` | Semáforo: Nodos colapsados (> 90%), SLA incumplido. |

**Micro-animaciones:** Los vuelos se grafican como vectores interconectados. Para evitar la carga masiva y concurrente que bloqueaba la UI principal (latencias reportadas), se emplea **Server-Sent Events (SSE)**. Esto permite renderizar trayectorias progresivas y transiciones CSS *ease-in-out* para el movimiento de paquetes, otorgando una experiencia de "simulación continua".

### 2.3 Vistas y Componentes Principales
**TO-BE:** el sistema no cambia de páginas pesadas; el flujo principal se concentra en el *Control Tower Dashboard* (Torre de Control) y en el módulo de experimentación.

**AS-IS (mapeo):** la Torre de Control se materializa en `frontend/src/App.jsx` y el módulo de experimentación vive en la ruta `/experiment` con `frontend/src/experiment/NumericExperimentDashboard.jsx`.

1. **Torre de Control — Vista en Vivo:** Ejecutada por el Hook `useControlTowerController.js`. Proporciona los controles de la simulación temporal (Día a Día o Colapso). Las rutas emergen asíncronamente en el mapa SVG basado en el flujo de SSE.
2. **NumericExperimentDashboard:** Dashboard dedicado a la ejecución masiva de escenarios (DoE). Incluye botones para forzar planificaciones y un área para descargar resultados operacionales en formato Excel `.xlsx`.
3. **AlgorithmComparisonPanel:** Tarjeta de resumen paramétrico. Compara el *fitness*, el tiempo de ejecución y las entregas a tiempo entre los algoritmos *Adaptive Large Neighborhood Search* (ALNS) y el *Algoritmo Genético Híbrido* (HGA).

### 2.4 Accesibilidad y Estándares
Todo diseño de componente debe cumplir con el estándar `61.std.gui.v01.md`:
* **WCAG 2.1 AA:** El contraste entre texto (ej. `#E8EDF2`) y fondos oscuros (`#1A2635`) supera el ratio 4.5:1.
* **Redundancia Visual:** El estatus crítico de un aeropuerto no se indica solo con color rojo (`#EF4444`), sino que se soporta numéricamente con el porcentaje de saturación (ej. `95%`) y tooltips nativos.
